# Fix "Rendered more hooks" Error

## Problem
The `NATIVE_USER` object in `AuthContext.tsx` (line 32-50) is a hand-rolled mock of Firebase's `User` interface. At runtime in Capacitor's WebView, this causes a silent throw that `ErrorBoundary` catches. The tree unmounts/remounts with a different hook count, producing error #310/#312.

## Fix
Replace the fake `NATIVE_USER` with a simple auth bypass that skips Firebase entirely on native.

### File: `src/contexts/AuthContext.tsx`
- Remove `NATIVE_USER` mock and all Firebase imports
- On native: set `user` to a plain `{ uid, email, displayName }` object (not Firebase `User`)
- Change `AuthContextType.user` to `User | SimpleUser | null` (union type)
- On web: keep Firebase Auth as-is

### File: `src/components/ProtectedRoute.tsx`
- No changes needed (already only checks `user` truthiness)

### File: `src/pages/Login.tsx`
- No changes needed (already only checks `user` truthiness)
