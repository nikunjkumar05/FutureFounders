# Deploy OpenWA Bot on Google Cloud (Free Tier)

## Overview
Run OpenWA + ngrok on a Google Cloud e2-micro instance (free for 12 months). Bot stays online even when your laptop is off.

## Prerequisites
- Google Cloud account (free tier)
- Credit card for verification (won't be charged)

## Step 1: Create GCP Instance
1. Go to console.cloud.google.com → Compute Engine → VM Instances
2. Create instance:
   - Name: `openwa-bot`
   - Machine type: `e2-micro` (free tier)
   - Region: `asia-south1` (Mumbai, closest to India)
   - Boot disk: Ubuntu 22.04
   - Firewall: Allow HTTP/HTTPS traffic
3. Note the external IP

## Step 2: SSH into Instance and Install Dependencies
```bash
# Update system
sudo apt update && sudo apt upgrade -y

# Install Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
sudo usermod -aG docker $USER

# Install Docker Compose
sudo apt install docker-compose -y

# Install Node.js (for OpenWA API)
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

# Install ngrok
curl -sSL https://ngrok-agent.s3.amazonaws.com/ngrok-v3-stable-linux-amd64.tgz | sudo tar -xz -C /usr/local/bin
```

## Step 3: Deploy OpenWA
1. Clone/copy OpenWA docker-compose to the instance
2. Start OpenWA container
3. Get ngrok authtoken (free from ngrok.com) and configure
4. Start ngrok tunnel
5. Scan QR code to link WhatsApp

## Step 4: Update Vercel Environment
1. Set `OPENWA_API_URL` to the ngrok tunnel URL (e.g., `https://xxxx.ngrok-free.app`)
2. Set `OPENWA_SESSION_ID` to the session ID from OpenWA
3. Set `OPENWA_API_KEY` to the API key
4. Set webhook URL in OpenWA config to point to Vercel webhook endpoint

## Step 5: Auto-Restart on Reboot
Create a systemd service to auto-start OpenWA + ngrok on instance restart:
```bash
# /etc/systemd/system/openwa-bot.service
[Unit]
Description=OpenWA WhatsApp Bot
After=docker.service

[Service]
Type=simple
WorkingDirectory=/home/user/openwa
ExecStart=/usr/bin/docker-compose up
ExecStop=/usr/bin/docker-compose down
Restart=always

[Install]
WantedBy=multi-user.target
```

## Files to Create/Modify
- `scripts/deploy-gcp.sh` — Setup script for new GCP instance
- `scripts/openwa-bot.service` — systemd service file
- `api/lib/openwa.ts` — No changes needed (reads env vars)

## Verification
1. SSH into instance → `docker ps` shows OpenWA running
2. `curl http://localhost:2785/api/health` returns `{"status":"ok"}`
3. Vercel webhook endpoint receives messages
4. Restart instance → bot auto-restarts
5. Shut down laptop → bot continues running
